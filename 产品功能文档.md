# 知识锁游戏产品功能文档

## 1. 项目定位

本项目是一个以“知识锁”为核心的 2D 平台跳跃解谜原型。玩家可以在系统层面获得能力，但游戏不直接解释完整用法；玩家需要通过关卡结构、失败反馈、环境暗示和自我尝试理解能力的隐藏用途。

当前故事基于《游戏故事V2》：主角是一块没有脸的混沌残片，在折叠、歪斜的世界中醒来，逐步获得四张面具，并在前进中理解自己不是占有世界，而是把歪掉的世界摆正、还回原样。

## 2. 当前版本目标

当前版本重点验证以下内容：

- 连续大地图房间结构，支持 20x20 普通房间和 40x40 大型/BOSS 房间。
- 无头盔状态、红色、绿色、白色、黑色四类能力的移动和解谜逻辑。
- 每个房间原则上需要存在无头盔可通关路线，同时也支持面具路线。
- 关卡编辑器可独立发送给同伴使用，并通过本地导出文件夹同步关卡。
- 编辑器导出的关卡数据可被试玩页面读取。
- 机关、危险、触发器、拾取物、表面元素、时序装置、灾厄喷口等功能格可在编辑器中配置。

当前版本不是最终美术版，也不是最终关卡版，主要用于验证机制、手感和关卡制作流程。

## 3. 运行方式

可玩版本推荐启动方式：

```text
启动游戏.bat
```

启动后打开：

```text
http://127.0.0.1:8010/index.html
```

不要直接双击 `index.html`。当前代码使用 ES modules，许多浏览器会阻止 `file://` 模式加载模块。

## 4. 独立关卡编辑器

编辑器单独放在：

```text
关卡编辑器-可发送版/
```

该文件夹应作为一个整体发送或提交，内部包含：

```text
editor.html
index.html
style.css
game-server.mjs
启动关卡编辑器.bat
关卡编辑器使用说明.txt
src/
关卡导出/
```

编辑器启动方式：

```text
关卡编辑器-可发送版/启动关卡编辑器.bat
```

启动后打开：

```text
http://127.0.0.1:8010/editor.html
```

编辑器保存规则：

- 编辑时会保存到浏览器 localStorage。
- 点击导出或试玩时，会写入 `关卡编辑器-可发送版/关卡导出/`。
- `room-XX.json` 表示单关数据。
- `all-levels.json` 表示全部关卡汇总。
- `map-layout.json` 表示大地图摆放位置。
- 把整个 `关卡编辑器-可发送版/` 发给同伴即可共享编辑器和当前导出的关卡。

编辑器支持：

- 单关编辑和大地图编辑。
- 20x20 / 40x40 房间尺寸。
- 在大地图中拖动房间，支持上下左右吸附拼接。
- 试玩本关和试玩总体。
- 试玩总体时，房间之间按大地图相邻关系无缝切换。
- 左侧按类别显示功能块：平台、表面、危险、触发器、机关、拾取物。
- 按住鼠标拖动连续绘制格子。
- 表面模式可放置瘟疫表面和闪电链桩。

## 5. 基础操作

| 操作 | 功能 |
| --- | --- |
| 左/右方向键 | 移动，或当前面具的左右能力 |
| 上方向键 | 跳跃，或当前面具的上方向能力 |
| 下方向键 | 下落平台，或当前面具的下方向能力 |
| Space | 未获得头盔时为无敌翻滚 |
| 长按 Space | 已获得头盔后打开面具选择，期间时间暂停 |
| Space + 右 | 选择红色面具 |
| Space + 下 | 选择绿色面具 |
| Space + 上 | 选择白色面具 |
| Space + 左 | 选择黑色面具 |
| R | 回到最近复活点 |
| M | 查看已经到达过的房间缩略图 |

面具规则：

- 玩家初始为无头盔状态。
- 解锁顺序为红、绿、白、黑。
- 一旦获得头盔，短按 Space 不再是翻滚，而是作为长按选择面具的功能键入口。
- 已获得的能力在死亡和复活后保留。

## 6. 角色形态

### 6.1 无头盔形态

- 左右移动。
- 单跳高度约 3.5 格。
- 最多二段跳。
- 下方向键可下落平台。
- 未获得头盔时，Space 为无敌翻滚。
- 翻滚受重力影响，但有额外初速度。
- 翻滚期间碰到致死装置不会死亡，并刷新翻滚能力。
- 翻滚刷新提示音最多 0.2 秒播放一次。
- 普通状态左右碰到致死表面时，有 0.1 秒保护期，在此期间按翻滚也算触墙跳。
- 空中贴墙触发翻滚时，角色会以翻滚相关速度向斜上方弹射；该弹射期间也视为翻滚。

### 6.2 红色面具

- 按方向键进行 4 格距离的脉冲冲刺。
- 冲刺先快后慢。
- 空中冲刺期间仍会缓慢受重力影响。
- 冲刺后出现 QTE 圆环。
- 在合适区间再次按方向键，可以继续冲刺。
- QTE 失败后会熄火并短暂无法行动。
- 红色冲刺可以杀死敌人。
- 红色冲刺不再破坏方块。
- 杀敌后可延长下一次 QTE 窗口，使下一次冲刺更容易成功。

### 6.3 绿色面具

- 实体状态下可左右移动和跳跃。
- 按下可在当前位置放置墓碑，并进入灵体状态。
- 灵体状态透明度较低，不受重力影响。
- 灵体状态可通过部分特殊通道或电网设计。
- 灵体不能触发需要重量的开关。
- 墓碑有重量，可以触发开关和易碎平台。
- 墓碑对除灵体以外的形态都有碰撞体积。
- 有墓碑时，灵体按上会传送回最新墓碑位置，并退出灵体状态。
- 多个墓碑之间形成连线。
- 非绿色形态触碰墓碑连线会死亡。
- 墓碑连线可以杀敌。
- 切换到其它面具后再切回绿色，会保持灵体/实体状态。

### 6.4 白色面具

- 无法普通跳跃。
- 会贴附在当前接触的地面、墙面或天花板上。
- 左右方向沿当前表面移动。
- 普通砖块表面移动速度为 5 格/秒。
- 瘟疫区域表面移动速度为 8 格/秒。
- 移动时只在走过的身后表面留下白色瘟疫。
- 白色瘟疫最长保留约 20 格。
- 非白色形态触碰白色瘟疫会死亡。
- 白色瘟疫可以杀敌。
- 按上发射钩锁。
- 钩锁最大长度为 6 格。
- 钩锁伸出速度为 20 格/秒。
- 按住上并命中目标表面后，会沿钩锁移动到命中的表面。

### 6.5 黑色面具

- 不能正常左右移动。
- 可以跳跃。
- 左方向键逆时针旋转整个世界。
- 右方向键顺时针旋转整个世界。
- 空中旋转世界时，旋转瞬间不受重力影响。
- 任意旋转世界后，切换到其它面具，关卡仍保持旋转后的状态。
- 屏幕重力始终向下。
- 玩家依靠跳跃、旋转世界和重力下落移动。
- 会侵蚀当前重力方向下方接触的普通方块。
- 不按下时约 6 秒侵蚀完当前方块。
- 按住下时约 1 秒侵蚀完当前方块。
- 方块侵蚀到 1/3 和 2/3 进度时出现裂纹反馈。
- 完全侵蚀后方块才失去碰撞体积。
- 黑色落在敌人上可以杀敌。

## 8. 表面瘟疫规则

表面瘟疫不是整格方块，而是绑定在格子边上的危险线段。

当前规则：

- 编辑器中只能在有实体的格子表面放置瘟疫。
- 空气格上不能直接放置瘟疫表面。
- 如果要给某格左表面放瘟疫，实际存储为左侧相邻格子的右表面。
- 如果要给某格下表面放瘟疫，实际存储为下方相邻格子的上表面。
- 该规范化用于避免左表面、下表面在显示和碰撞中反复出错。
- 规范化后的存储格即使是空气，也可以保留，因为它代表相邻实体格的边。
- 如果一个瘟疫表面两侧都没有任何可附着格，会在编辑器加载/去重、试玩数据读取和正式解析时被过滤掉。
- 转角区域两端各 5% 不生成瘟疫表面，避免转角重叠和误判。
- 非白色形态触碰白色瘟疫会死亡。
- 白色形态免疫白色瘟疫。

## 9. 闪电链规则

- 闪电链端点放置在格子表面，而不是整格中心。
- 第一次点击放置第一个桩，第二次点击放置第二个桩并连接成链，后续端点依次连接。
- 闪电链可以首尾相连。
- 退出闪电链编辑后，再次进入会开始一条新的链。
- 闪电桩本体不造成伤害，只有中间线条造成伤害。
- 绿色形态免疫闪电链。
- 普通形态和其它非绿色形态触碰闪电链会死亡。
- 墓碑连线和闪电链使用同一类高亮颜色。

## 10. 开关、触发器与可控对象

开关或触发器放置前，必须先指定控制目标。可控目标包括：

- 门。
- 尖刺。
- 金币。
- 单根闪电链。
- 单个瘟疫表面。
- 灾厄喷口。

控制规则：

- 如果多个开关控制同一个对象，需要全部满足后对象才改变状态。
- 门：开/关，打开时有消失动画，第一帧即关闭碰撞。
- 尖刺：出现/消失。
- 金币：出现/消失，默认可配置。
- 闪电链：出现/消失。
- 瘟疫表面：出现/消失。
- 灾厄喷口：启用/禁用。
- 单次按压开关触发后永久保持。
- 多次按压开关每次重新触碰都会切换。
- 隐形触发器外观为空地，但作用类似开关。

## 11. 时序装置

时序装置用于按固定间隔依次触发一个或多个目标。

可配置项：

- 触发模式：持续触发或条件触发。
- 条件类型：拾取物或触发器。
- 间隔时间：0.2 秒到 3 秒。
- 每步持续时间。
- 是否循环。
- 目标范围：瘟疫、闪电链、尖刺、灾厄喷口等可控对象。

## 12. 灾厄喷口

灾厄喷口是永久危险发射机关。

可配置项：

- 发射危险物：尖刺、瘟疫、闪电。
- 发射物大小：按格子边长作为单位。
- 发射/休息占比。
- 发射周期。
- 发射速度：按格/秒作为单位。
- 发射方向：拖动选择 8 方向。
- 追踪时间：0 表示直线，0.5 秒到 3 秒表示追踪玩家。

子弹伤害规则：

- 尖刺子弹会杀死所有形态。
- 瘟疫子弹不会杀死白色形态。
- 闪电子弹不会杀死绿色形态。

## 13. 复活、死亡与状态保留

死亡来源包括：

- 尖刺。
- 闪电链。
- 非白色形态触碰白色瘟疫。
- 非绿色形态触碰绿色墓碑连线。
- 敌人接触。
- 灾厄喷口发射出的有效危险子弹。
- 后续关卡扩展的致死装置。

边界规则：

- 掉出房间边界本身不应致死，除非关卡明确放置了致死装置。

复活规则：

- 玩家触碰复活点后，会更新最近复活点。
- 复活点可跨关卡保存。
- 如果在第四关摸到复活点，然后回到第三关死亡，应在第四关复活。
- 当前可复活的小旗子保持绿色。
- 死亡后保留已经获得的头盔和能力。
- 死亡后保留当前已获得能力状态，不回到一开始的无头盔状态。

## 14. 房间与大地图

- 普通房间为 20x20。
- BOSS/大型房间为 40x40。
- 大地图可以无限拖动和拼接。
- 房间在大地图中通过相邻边连接。
- 跨房间时保持角色状态、速度和相对位置。
- 大房间连接小房间时，不按比例映射，而是按实际贴合区域换算：例如 20 高的房间贴在 40 高房间的下半，则左右切换时使用 40 房间内的当前高度；贴在上半则减去 20 格高度。
- 已进入过的新关卡才会出现在 M 键缩略图中。

## 16. 技术结构

项目使用原生 HTML、CSS、JavaScript 和 Canvas。

可玩版本核心文件：

```text
index.html
style.css
README.txt
启动游戏.bat
game-server.mjs
src/
```

独立编辑器核心文件夹：

```text
关卡编辑器-可发送版/
```

主要职责：

| 文件 | 职责 |
| --- | --- |
| `src/main.js` | 主循环、输入、HUD、存档、死亡、切房、拾取、机关和危险判定 |
| `src/constants.js` | 尺寸、速度、颜色、面具名称等常量 |
| `src/world.js` | 房间拓扑、房间生成、关卡字符解析 |
| `src/player.js` | 玩家初始状态 |
| `src/physics.js` | 碰撞、平台、地面检测、白色贴面检测、旋转变换 |
| `src/mechanics.js` | 无头盔、红、绿、白、黑的能力逻辑 |
| `src/render.js` | Canvas 绘制、角色、方块、污染、UI 叠层 |
| `关卡编辑器-可发送版/editor.html` | 独立关卡编辑器 UI、编辑逻辑、大地图编辑和试玩入口 |
| `关卡编辑器-可发送版/game-server.mjs` | 编辑器本地服务器、关卡导出/删除 API |

## 17. 当前已知风险

- 白色贴面、凸角转面、钩锁落点和污染边界仍需持续测试。
- 表面瘟疫的视觉位置与判定需要继续校验，尤其是复杂转角与大房间连接处。
- 黑色旋转世界后的边界、存档点和房间切换仍需在复杂地图中验证。
- 当前关卡不是最终关卡，后续会由编辑器设计更多房间。
- 当前美术仍为程序图形，后续需要替换为正式贴图、动画和音效。
