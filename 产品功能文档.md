# 无面知识锁 Demo 产品功能文档

## 1. 项目定位

本项目是一个以“知识锁”为核心的 2D 平台跳跃解谜原型。

知识锁的目标是：玩家从系统层面已经拥有或可以获得能力，但游戏不直接解释完整用法；玩家需要通过关卡结构、失败反馈、环境暗示和自我尝试，逐步理解能力的隐藏用途。

当前故事基于《游戏故事V2》：主角是一块没有脸的混沌残片，在折叠、歪斜的世界中醒来。它逐步拾起四张由相柳、夔、精卫、烛龙残余凝成的面具，并在三道质问中理解自己的使命：不是占有世界，而是把歪掉的世界摆正、还回原样。

## 2. 当前版本目标

当前版本重点验证：

- 连续大地图房间结构。
- 无面形态的基础平台跳跃和无头盔路线。
- 红、绿、白、黑四张面具的移动能力。
- 每个房间内的机关、危险、道具、存档和复活规则。
- 玩家不拿面具也能尝试通关；拿面具后可通过能力组合解决更复杂路径。
- 编辑器导出的关卡数据能被正式试玩读取。

当前版本不是最终美术版，也不是最终关卡版。它主要用于验证机制是否有趣、是否可控、是否能支撑大量关卡设计。

## 3. 运行方式

推荐启动方式：

```text
启动游戏.bat
```

该脚本会启动本地服务器并打开：

```text
http://127.0.0.1:8010/index.html
```

不要直接双击 `index.html`。当前代码使用 ES modules，许多浏览器会阻止 `file://` 模式加载模块。

## 4. 基础操作

| 操作 | 功能 |
| --- | --- |
| 左 / 右方向键 | 移动，或当前面具的左右能力 |
| 上方向键 | 跳跃，或当前面具的上方向能力 |
| 下方向键 | 下落平台，或当前面具的下方向能力 |
| Space | 无面且未获得头盔时为翻滚 |
| 长按 Space | 已获得面具后打开面具选择，期间时间暂停 |
| Space + 右 | 选择红色夔面 |
| Space + 下 | 选择绿色精卫面 |
| Space + 上 | 选择白色相柳面 |
| Space + 左 | 选择黑色烛龙面 |
| R | 回到最近存档点 |
| M | 查看已到达房间缩略图 |

面具规则：

- 玩家一开始是无面形态。
- 红色面具是第一个可获得能力。
- 后续能力按当前设计顺序逐步获得：红、绿、白、黑。
- 一旦获得头盔，短按 Space 不再是翻滚，而是作为长按选择面具的功能键入口。
- 已获得的能力会在死亡和复活后保留。

## 5. 世界与叙事结构

当前故事分为三层叙事目标：

- 第一幕：残片逐步拾起四张面具，想起“我是什么”。
- 第二幕：守阈者和钦䲹逼问残片，促使它理解“我为什么前进”。
- 第三幕：鼓试图占有世界，残片最终选择还原世界而不是占有世界。

四张面具的叙事含义：

| 面具 | 灵感 | 关键词 | 机制气质 |
| --- | --- | --- | --- |
| 红色夔面 | 雷泽、鼓点、搏动 | 踔、战争、节奏 | 冲刺、判断、杀敌 |
| 绿色精卫面 | 衔石、记忆、返回 | 衔、死亡、留痕 | 墓碑、灵体、回到来处 |
| 白色相柳面 | 毒泽、蔓延、聚合 | 蔓、瘟疫、表面 | 贴附、钩锁、污染 |
| 黑色烛龙面 | 钟山、昼夜、朝向 | 转天、饥荒、姿态 | 旋转世界、侵蚀地形 |

## 6. 角色形态

### 6.1 无面形态

颜色：灰蓝。

能力：

- 左右移动。
- 单跳高度约 3.5 格。
- 最多二段跳。
- 下方向键可下落平台。
- 未获得头盔时，Space 为无敌翻滚。
- 翻滚会受到重力影响，但具有额外水平初速度。
- 翻滚期间碰到致死装置时不会死亡，并会刷新翻滚冷却。

设计目标：

无面形态是表面上的普通平台跳跃角色，但仍应有知识锁式技巧。所有房间原则上都应存在无面可通关路线，只是可能需要更高操作或更绕的路径。

### 6.2 红色夔面

核心关键词：冲刺、节奏、QTE、杀敌。

能力：

- 按方向键进行 4 格距离的脉冲冲刺。
- 冲刺先快后慢。
- 空中冲刺期间仍会受到轻微重力影响。
- 冲刺后出现 QTE 圆环。
- 在合适区间再次按方向键，可以继续冲刺。
- QTE 失败后会熄火并短暂不能行动。
- 红色冲刺可以杀死敌人。
- 红色冲刺不再破坏方块。
- 杀敌后可以让下一次 QTE 更宽松。

设计目标：

红色夔面是“冲”的能力。它适合穿越危险长廊、在空中精确选择方向、快速击杀敌人，以及在有限窗口内完成连续行动。

### 6.3 绿色精卫面

核心关键词：墓碑、灵体、记忆、返回。

能力：

- 实体状态下可以左右移动和跳跃。
- 按下会在当前位置放置墓碑，并进入灵体状态。
- 灵体状态透明度较低，不受重力影响。
- 灵体状态可以通过部分特殊通道或电网设计。
- 灵体不能触发需要重量的开关。
- 墓碑可以触发需要重量的开关。
- 有墓碑时，灵体按上会传送回最新墓碑位置，并退出灵体。
- 多个墓碑之间会形成连线。
- 非绿色形态触碰墓碑连线会死亡。
- 墓碑连线可以杀敌。
- 切换到其他面具后再切回绿色，会保留灵体状态。

设计目标：

绿色精卫面是“躲”和“记得”的能力。它适合做延迟返回、危险区侦察、开关与身体位置分离、追逐逃脱、通过灵体专属通道等谜题。

### 6.4 白色相柳面

核心关键词：贴附、表面移动、钩锁、瘟疫。

能力：

- 无法普通跳跃。
- 会贴附在当前接触的地面、墙面或天花板上。
- 左右方向会沿当前表面移动。
- 在普通砖块表面移动速度为 5 格/秒。
- 在瘟疫区域表面移动速度为 8 格/秒。
- 移动时只在走过的身后表面留下白色瘟疫。
- 白色瘟疫最长保留约 20 格。
- 非白色形态触碰白色瘟疫会死亡。
- 白色瘟疫可以杀敌。
- 按上发射钩锁。
- 钩锁最大长度为 6 格。
- 钩锁伸出速度为 20 格/秒。
- 按住上并命中目标表面后，会沿钩锁移动到命中的表面。

设计目标：

白色相柳面是“等”和“蔓延”的能力。它适合沿表面定位、等待时机、制造危险路径、利用钩锁换面，以及让玩家理解“走过的路会改变后续空间”。

### 6.5 黑色烛龙面

核心关键词：旋转世界、重力、侵蚀、创造路径。

能力：

- 不能正常左右移动。
- 可以跳跃。
- 左方向键逆时针旋转整个世界。
- 右方向键顺时针旋转整个世界。
- 在空中旋转世界时，旋转瞬间不受重力影响。
- 任意旋转世界后，切换到其他面具，关卡仍保持旋转后的状态。
- 屏幕重力始终向下。
- 玩家依靠跳跃、旋转世界和重力下落移动。
- 会侵蚀当前重力方向下方接触的普通方块。
- 不按下时约 6 秒侵蚀完当前方块。
- 按住下时约 1 秒侵蚀完当前方块。
- 方块侵蚀到 1/3、2/3 进度时出现裂纹反馈。
- 完全侵蚀后方块才失去碰撞体积。
- 黑色落在敌人上可以杀敌。

设计目标：

黑色烛龙面是“创造”的能力。它把移动问题变成改变世界姿态和改造地形的问题，适合用来打开通路、改变其他面具的行动环境，以及制造可下落或可钩锁的新结构。

## 7. 功能格与关卡字符

正式试玩会解析以下功能格。编辑器导出的同名字符应在试玩中具有相同效果。

| 字符 | 名称 | 功能 |
| --- | --- | --- |
| `.` | 空地 | 可通行区域 |
| `#` | 普通方块 | 实体碰撞；可被黑色侵蚀 |
| `=` | 上平台高 | 朝上的高半砖平台 |
| `-` | 上平台低 | 朝上的低半砖平台 |
| `U` | 下平台 | 朝下的半砖平台 |
| `L` | 左平台 | 朝左的半砖平台 |
| `J` | 右平台 | 朝右的半砖平台 |
| `H` | 隐藏砖 | 绿色灵体状态可见 |
| `!` | 尖刺 | 致死危险 |
| `~` | 电网 | 致死危险；绿色灵体可免疫 |
| `M` | 敌人 | 接触通常致死；可被红冲刺、白瘟疫、绿墓碑线、黑踩踏杀死 |
| `K` | 按压开关 | 玩家实体或墓碑可触发；绿色灵体不能触发 |
| `>` | 右扳机 | 只能从右侧推到左侧时切换开关状态 |
| `<` | 左扳机 | 只能从左侧推到右侧时切换开关状态 |
| `^` | 上扳机 | 只能从上侧推到下侧时切换开关状态 |
| `v` | 下扳机 | 只能从下侧推到上侧时切换开关状态 |
| `S` | 旧版扳机 | 兼容旧关卡，等同右扳机 |
| `D` | 门 | 由开关控制开闭 |
| `F` | 复活点 | 玩家触碰后，死亡或按 R 会回到此处 |
| `G` | 绿色能力 | 拾取后解锁绿色精卫面 |
| `R` | 红色能力 | 若尚未获得头盔，拾取后获得头盔并解锁红色夔面 |
| `W` | 白色能力 | 拾取后解锁白色相柳面 |
| `B` | 黑色能力 | 拾取后解锁黑色烛龙面 |
| `A` | 钩锁锚点 | 白色钩锁相关视觉/机制扩展点 |
| `P` | 格内瘟疫 | 预设白色瘟疫危险 |

表面瘟疫：

- 编辑器还支持给某个方块的某个表面放置瘟疫。
- 表面瘟疫不是一个整格方块，而是绑定在方块边上的危险线段。
- 如果瘟疫在某个方块上表面，玩家位于其上方格并接触该表面时会受到影响。
- 只有白色形态免疫白色瘟疫。

## 8. 存档、死亡与状态保留

死亡来源包括：

- 尖刺。
- 电网。
- 非白色形态触碰白色瘟疫。
- 非绿色形态触碰绿色墓碑线。
- 敌人接触。
- 后续关卡扩展的致死装置。

掉出房间边界本身不应致死，除非关卡明确放置了致死装置。

复活规则：

- 玩家触碰小旗子或复活点后，会更新最近存档点。
- 死亡或按 R 后，玩家回到最近存档点。
- 已获得的头盔和能力会保留。
- 已升起的旗子状态会保留。
- 当前面具状态会随存档点记录。

## 10. 技术结构

项目使用原生 HTML、CSS、JavaScript 和 Canvas。

可玩版本核心文件：

```text
index.html
style.css
README.txt
启动游戏.bat
game-server.mjs
src/
  main.js
  constants.js
  world.js
  player.js
  physics.js
  mechanics.js
  render.js
```

职责划分：

| 文件 | 职责 |
| --- | --- |
| `src/main.js` | 主循环、输入、HUD、存档、死亡、切房、拾取和机关 |
| `src/constants.js` | 尺寸、速度、颜色、面具名称等常量 |
| `src/world.js` | 房间拓扑、房间生成、关卡字符解析 |
| `src/player.js` | 玩家初始状态 |
| `src/physics.js` | 碰撞、平台、地面检测、白色贴面检测、旋转变换 |
| `src/mechanics.js` | 无面、红、绿、白、黑的能力逻辑 |
| `src/render.js` | Canvas 绘制、角色、方块、污染、UI 叠层 |

## 11. 当前已知风险

- 白色相柳面的贴附、凸角转面、下表面、钩锁落点和污染边界仍需重点测试。
- 表面瘟疫和角色碰撞边界需要持续校验，避免视觉位置与判定不一致。
- 黑色旋转世界后的边界、存档点和房间切换仍需在复杂地图中验证。
- 当前关卡不是最终关卡，后续会由编辑器设计更多房间。
- 当前美术仍为程序图形，后续需要替换为正式贴图、动画和音效。

## 12. 一句话描述

一个以“无面残片拾起四张神话面具”为叙事核心的知识锁平台跳跃解谜原型：玩家可以不用面具通关，也可以逐步获得红、绿、白、黑四种极端移动方式，在折叠世界中通过尝试理解机制，并最终把歪掉的世界还回原样。
