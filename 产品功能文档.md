# 四骑士知识锁 Demo 产品功能文档

## 1. 项目定位

本项目是一个以“知识锁”为核心的 2D 平台跳跃解谜 Demo。

知识锁的设计目标是：玩家从机制上已经拥有能力，但游戏不直接解释完整用法；玩家需要通过关卡结构、失败反馈和自我尝试，逐步理解能力的隐藏用途。

当前主题灵感来自“天启四骑士”。玩家可以选择不使用头盔，以灰蓝无头盔形态通关；也可以在第 2 关拾取头盔后切换四种骑士能力。

## 2. 当前版本目标

当前版本重点验证：

- 连续大地图房间结构
- 无头盔基础平台跳跃
- 可选头盔路线
- 白色骑士的表面移动、钩锁、污染区域
- 红色骑士的脉冲位移与 QTE
- 绿色骑士的墓碑、灵体、回溯
- 黑色骑士的世界旋转与侵蚀
- 小旗子存档点与死亡复活

当前版本不是最终美术版，也不是完整关卡版。它主要用于验证核心机制是否有趣、是否可控、是否能发展成关卡。

## 3. 启动与运行

推荐启动方式：

双击项目根目录下的：

```text
启动游戏.bat
```

该脚本会启动本地服务器并打开：

```text
http://127.0.0.1:8010/index.html
```

不要直接双击 `index.html`。当前代码使用 ES modules，许多浏览器会阻止 `file://` 模式加载模块。

## 4. 基础操作

| 操作 | 功能 |
| --- | --- |
| 左 / 右方向键 | 移动 |
| 上方向键 | 跳跃 |
| 下方向键 | 下落平台，或部分骑士特殊能力 |
| 长按 Space | 打开骑士选择，期间时间暂停 |
| Space + 上 | 选择白色骑士 |
| Space + 右 | 选择红色骑士 |
| Space + 左 | 选择黑色骑士 |
| Space + 下 | 选择绿色骑士 |
| R | 回到最近触碰过的小旗子存档点 |
| N / P | 调试用切换房间 |

头盔规则：

- 第 2 关可以拾取头盔。
- 不拾取头盔也可以继续前进。
- 拾取头盔后，可以长按 Space 切换骑士。
- 一旦戴上头盔，就不能回到无头盔形态。

## 5. 地图与关卡结构

当前地图由 50 个房间组成。

设计原则：

- 房间不是传送门式终点，而是连续大地图。
- 玩家走到房间边界会切换到相邻房间。
- 可以反向走回之前房间。
- 地图包含主路、支路、回环和捷径。
- 每个房间入口附近放置一个小旗子存档点。

存档点规则：

- 玩家触碰小旗子后，小旗子升起。
- 升起状态永久保留。
- 死亡或按 R 后，玩家回到最近触碰过的小旗子。
- 复活只考虑最近触碰过的旗子，不考虑视觉上最近的旗子。

## 6. 角色形态

### 6.1 无头盔形态

颜色：灰蓝色。

基础能力：

- 左右移动
- 跳跃
- 二段跳
- 下落平台

设计目标：

无头盔形态是表面上的普通平台跳跃角色，但应保留知识锁式隐藏技巧。当前版本已有基础操作，后续可以继续加强无头盔路线，让它能以更高技巧通过同一批谜题。

### 6.2 白色骑士：瘟疫

核心关键词：贴附、表面移动、污染、钩锁。

当前能力：

- 无法普通跳跃。
- 会贴附在当前接触的地面、墙面或天花板上。
- 左右方向会沿着当前表面移动。
- 移动时会在走过的表面留下白色污染区域。
- 非白色形态碰到白色污染区域会死亡。
- 按上会向当前表面的“正上方”发射钩锁。
- 如果按住上并命中目标表面，会沿钩锁移动到命中的方块表面。

污染区域规则：

- 表现上是小斑点和滴落效果。
- 计算上不是无限叠加斑点，而是记录表面污染区间。
- 同一段表面来回移动只会合并或扩展区间，不会持续增加大量对象。
- 左右接触面也会登记污染。
- 钩锁拉到新方块后，会给命中的面登记污染。

设计目标：

白色骑士应该像“沿建筑表面蔓延的瘟疫”。污染既是危险，也是空间记忆。玩家需要理解自己制造的路线会改变后续行动的风险。

### 6.3 红色骑士：战争

核心关键词：脉冲位移、节奏、QTE、爆破。

当前能力：

- 按方向键会进行四格距离的脉冲位移。
- 位移先快后慢。
- 空中位移时仍会受到轻微重力影响。
- 位移后出现 QTE 圆环。
- 在合适区间再次按方向键，可以继续位移。
- 失败后会熄火并短暂不能行动。
- 向下位移可以破坏裂纹方块。

设计目标：

红色骑士是节奏型移动角色。玩家需要理解位移距离、QTE时机、方向选择和破坏地形之间的关系。

### 6.4 绿色骑士：死亡

核心关键词：墓碑、灵体、回溯、隐藏通道。

当前能力：

- 普通状态下可以左右移动和跳跃。
- 按下会在当前位置放置墓碑，并进入灵体状态。
- 灵体状态透明度较低。
- 灵体状态不受重力影响。
- 灵体状态可以看到隐藏砖块。
- 灵体状态按上，会传送回最新墓碑位置，并退出灵体。
- 墓碑之间会形成线。
- 非绿色形态触碰墓碑线会死亡。

设计目标：

绿色骑士提供“空间标记 + 返回”的解谜方式。玩家要理解什么时候留下墓碑、什么时候进入灵体、什么时候回溯。

### 6.5 黑色骑士：饥荒

核心关键词：无法主动移动、世界旋转、侵蚀。

当前能力：

- 不能正常左右移动。
- 可以跳跃。
- 左方向键逆时针旋转整个世界。
- 右方向键顺时针旋转整个世界。
- 在空中旋转世界时，旋转瞬间不受重力影响。
- 任意旋转世界后，切换到其他骑士，关卡仍保持旋转后的状态。
- 屏幕重力始终向下。
- 玩家依靠跳跃、旋转世界和重力下落移动。
- 会缓慢侵蚀下方方块。
- 按下可以加速侵蚀。

设计目标：

黑色骑士把移动问题变成“改变世界姿态”的问题。玩家不能左右行走，但可以跳跃，并通过旋转场景、重力下落和侵蚀地形创造路径。旋转后的世界状态会影响其他骑士。

## 7. 当前房间教学节奏

当前前 10 关设计意图：

| 房间 | 教学重点 |
| --- | --- |
| 01 | 左右移动、跳跃、二段跳、白色污染死亡陷阱 |
| 02 | 可选头盔拾取 |
| 03 | 无头盔路线与头盔路线并存 |
| 04 | 白色贴附表面移动 |
| 05 | 白色污染区域 |
| 06 | 白色钩锁 |
| 07 | 红色四格脉冲 |
| 08 | 红色 QTE 连续位移 |
| 09 | 绿色墓碑、灵体、隐藏砖 |
| 10 | 黑色旋转世界与侵蚀 |

后续 11 到 50 关目前主要用于测试连续地图、支路、回环和机制组合，还不是最终关卡设计。

## 8. 失败与死亡

当前死亡来源：

- 掉出房间有效区域
- 非白色形态触碰白色污染
- 非绿色形态触碰绿色墓碑线
- 其他后续关卡机关可以继续扩展

死亡后：

- 回到最近触碰过的小旗子。
- 保留已升起旗子状态。
- 保留头盔拾取状态。

## 9. 当前技术结构

项目使用原生 HTML、CSS、JavaScript 和 Canvas。

文件结构：

```text
index.html
style.css
README.txt
启动游戏.bat
src/
  main.js
  constants.js
  world.js
  player.js
  physics.js
  mechanics.js
  render.js
```

职责划分：

| 文件 | 职责 |
| --- | --- |
| `src/main.js` | 主循环、输入、HUD、存档、死亡、切房 |
| `src/constants.js` | 尺寸、速度、颜色、骑士名称等常量 |
| `src/world.js` | 房间拓扑、房间生成、关卡字符解析 |
| `src/player.js` | 玩家初始状态 |
| `src/physics.js` | 碰撞、地面检测、白色贴面检测、旋转变换 |
| `src/mechanics.js` | 无头盔、白、红、绿、黑的能力逻辑 |
| `src/render.js` | Canvas 绘制、角色、方块、污染、UI叠层 |

## 10. 当前已知风险

### 10.1 白色骑士仍需重点测试

白色骑士是当前最复杂的形态，尤其是：

- 地块下表面吸附
- 转角切换
- 钩锁命中后的新表面吸附
- 污染区域是否与实际接触面一致
- 左右侧面污染是否可靠

### 10.2 关卡仍是原型关卡

当前 50 房间地图是机制验证地图，不是最终体验地图。后续需要逐关调整节奏、难度、空间结构和多解路线。

### 10.3 美术表现仍为程序图形

当前所有图形由 Canvas 程序绘制，没有正式角色动画、贴图、粒子或音效。

## 11. 后续优先级建议

建议后续优先级：

1. 稳定白色骑士的贴附、钩锁、污染逻辑。
2. 重做前 10 关，确保每关明确教学一个新知识点。
3. 为无头盔形态补足更有趣的知识锁技巧。
4. 调整红色 QTE 的节奏、宽容区间和反馈。
5. 完善绿色灵体的隐藏通道和墓碑线谜题。
6. 完善黑色旋转世界的专属关卡。
7. 给每种骑士增加更清晰的视觉反馈。
8. 增加音效和失败反馈。
9. 做导出包和试玩说明。

## 12. 当前版本一句话描述

一个以天启四骑士为灵感的知识锁平台跳跃解谜原型：玩家可以选择不用头盔，也可以拾取头盔切换四种极端移动方式，在连续大地图中通过尝试理解机制。
