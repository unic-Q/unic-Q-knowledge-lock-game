# 当前问题备注

## 回退说明

本包已回退到“白色骑士每帧先找附近能贴的面，贴上去之后沿这个面的切线移动；移动后如果身体和砖块重叠则按重叠方向换面；否则在中心越过当前砖块边界时用 exteriorCornerSurface() 猜外拐角”的版本附近。

也就是说，最后一次尝试加入的“优先维持上一帧吸附面”和“同法线续接面搜索”已经撤回。

## 当前主要问题

白色瘟疫骑士在贴着平台的左边缘或下边缘移动时，尤其是在任意方格的拐角处尝试进入下一个方格，角色可能突然消失、卡死或吸附到不符合预期的表面。

这不是单纯的渲染问题。根据调试状态，角色坐标和 whiteSurface 状态会进入不稳定状态，表现出来像是角色消失或被吸到错误位置。

## 目前怀疑的原因

1. `findWhiteSurface(state)` 是全局找最近可贴面，砖块接缝和拐角附近会同时存在多个距离很近的候选面。
2. 在左边缘和下边缘，角色中心、身体矩形、砖块边界之间的判断更容易出现“上一帧还在当前面，下一帧被判到隔壁块或另一面”的跳变。
3. `snapWhiteToSurface()` 会直接把角色坐标贴到候选面的标准位置，如果候选面选错，就会造成明显坐标跳变。
4. `resolveWhiteOverlap()` 和 `exteriorCornerSurface()` 都是在移动之后修正面；如果前一步选面已经错了，后续修正可能会进一步把角色带到错误面。
5. 污染区域现在按格子面记录，但白色移动的表面选择仍然是连续坐标和矩形碰撞混合判断，两者在转角处还没有完全统一。

## 可能解决方法

方案 A：保留最早逻辑，但约束候选面

在 `findWhiteSurface()` 中加入更严格的优先级：

- 当前吸附面的同一块优先。
- 当前吸附面的同法线相邻块优先。
- 只有当前面明显不可用时，才允许切到垂直面。
- 在砖块接缝处不要因为另一个面的距离相同或略小就立即切面。

优点：改动较小，能保留当前手感。

风险：如果规则写得太细，会继续出现边缘特判越来越多的问题。

方案 B：白色骑士改成“面图状态机”

把可贴表面建模为格子面：

- 状态 = 当前格子 + 当前面。
- 沿面移动时只改变面上的进度。
- 进入下一格时明确切到相邻格子的同一面。
- 到达凸角/凹角时按拓扑关系切到新面。

优点：逻辑稳定，污染区域也天然绑定格子面。

风险：需要重写白色移动核心，但长期更可控。

方案 C：连续坐标移动 + 局部候选过滤

继续使用连续坐标，但每帧只检测角色附近极小范围内、且和上一帧运动方向一致的表面。切面必须满足“距离、方向、接触长度、上一帧状态”共同条件。

优点：比方案 B 改动小，可能较快修复。

风险：仍然可能有复杂边界问题。

## 建议下一步

建议优先尝试方案 B。白色骑士的玩法核心就是沿表面移动和在表面留下污染，如果继续用普通矩形碰撞去猜表面，左边缘、下边缘、凸角、凹角、接缝都会持续出问题。

如果短期只想让 demo 可玩，可以先做方案 A：限制 `findWhiteSurface()` 在接缝处不要抢面，并给切面动作加明确条件。
