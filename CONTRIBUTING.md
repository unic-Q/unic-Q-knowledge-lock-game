# 协作流程

这个项目采用“主控合并”的协作方式：其他人可以在自己的分支开发，但进入主分支前必须通过 Pull Request。

## 推荐流程

1. 从 `main` 拉最新代码。
2. 为每个任务新建分支，例如 `feature/white-movement-fix`、`level/room-03-pass`。
3. 本地测试能打开游戏后提交。
4. 推送自己的分支并创建 Pull Request。
5. 由项目主控检查后决定是否合并。

## 分支约定

- `main`：稳定版本，只放已经决定保留的内容。
- `feature/*`：机制、玩法、界面功能。
- `fix/*`：bug 修复。
- `level/*`：关卡调整。
- `docs/*`：文档和说明。

## 提交前检查

- `启动游戏.bat` 能正常打开游戏。
- 不提交本地临时文件、压缩包、调试状态文件。
- 在 PR 里说明改了什么、为什么改、怎么试玩。
