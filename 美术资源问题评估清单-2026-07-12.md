# 美术资源问题评估清单

评估日期：2026-07-12  
评估范围：当前前端实际使用的运行时角色 Atlas、Tile、场景物件、特效与界面资源。  
检查规模：35 套角色动画，共 246 帧。  

## 一、评估结论

当前前端已经完成主要美术资源接入，但资源本身和动画播放规范仍存在以下五类问题：

1. 部分角色帧的白底、白条和地面残留没有清理干净。
2. 部分动作帧的角色基线和实际内容尺寸不一致。
3. 运行时以整张帧画布高度缩放，容易放大素材尺寸差异。
4. 少数关键游戏对象仍缺正式美术，当前使用程序图形占位。
5. 已有动画资源没有全部进入实际状态机，影响动作完整性。

## 四、统一导出规范建议

后续所有角色动画建议执行同一套交付规范：

- 文件必须使用带 Alpha 通道的 PNG，不得保留白色或灰色底板。
- 同一角色的基础视觉身高必须统一，动作特效不能改变人物缩放基准。
- 每帧都要提供统一的脚底中心 Pivot；腾空动作需要明确空中位移由素材还是程序控制。
- 人物、武器、藤蔓、冲击波等特效应尽量分层，避免特效扩大整帧画布后改变角色尺寸。
- Atlas 边缘保留 1–2px 透明 Padding，避免纹理采样串色。
- JSON 至少记录 `frame`、`pivot`、`displayScale`、`fps`、`loop` 和动画事件。
- 技能流程统一按 `start / active / end` 组织；复杂动画需要标记可循环区段。
- 导出后必须在深色、浅色和棋盘格三种背景上检查白边。
- 每套动画交付前至少播放三个完整循环，检查闪白、跳脚、体型突变和特效截断。

## 五、建议修复顺序

1. 修复精卫分魂、无面跳跃、无面跑步、精卫跑步、相柳受击的白底残留。
2. 修复无面跳跃和相柳技能持续动画的脚底基线。
3. 统一角色动画的 `displayScale`、内容包围盒和 Pivot 规范。
4. 补齐受击、死亡以及技能 start、active、end 的播放状态。
5. 补敌人、夔相种和红色破裂块三项核心缺失素材。
6. 补金币、旗帜、墓碑、能力拾取物和程序绘制特效。
7. 最后统一暂停、死亡、通关、地图和四相选择等界面包装。

## 六、验收标准

- 所有运行时角色帧在棋盘格背景下无白底、白条和矩形残留。
- 同一角色在 idle、run、jump、hit 和 ability 之间切换时，基础体型没有肉眼可见的缩放跳变。
- 地面动作的脚底基线误差不超过 2px；需要腾空的动作使用明确的位移规范。
- 所有已交付动画都能在前端状态机中实际触发。
- 敌人、夔相种和红色破裂块不再使用程序占位图。
- 场景核心物件和特效与当前神话水墨像素风格保持一致。
